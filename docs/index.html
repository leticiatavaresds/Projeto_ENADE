Introdução¶

O escopo do projeto dividido em trs partes. Primeiramento, construdo um Data Warehouse seguindo uma modelagem dimensional estrela com os microdados do ENADE do anos de 2017, 2018 e 2019 disponibilizados no site do INEP. A partir de consultas ao Data Warehouse elaborado, ento realizada uma Anlise Exploratria para por fim construir um modelo de aprendizagem de mquina com a finalidade de predizer o desempenho de um aluno com base em informaes preenchidas nas provas como dados da instituio, caractersticas e perfil socioeconmico do aluno. As bases de dados importadas possuem 169 variveis, mas no presente projeto algumas foram descartadas. Todo o trabalho est disponvel em um repositrio do Github [1].

Microdados ENADE¶

O ENADE, sigla para Exame Nacional de Desempenho de Estudantes, uma exame anual que tem como objetivo avaliar o conhecimento obtidos em relao aos contedos previstos na grade curricular do respectivo curso de graduao dos estudantes acadmicos que esto prestes a se formar. Dessa forma, todo ano, alunos concluintes do ensino superior so submetidos a uma prova de conhecimento geral e especfico composta por questes objetivas e discursivas. Alm das questes para avaliar o conhecimento do estudante, as provas tambm apresentam questionrios para obter informaes socionmicas e experincias acadmicas do estudante, alm de seu preparo para a prova, percepo sobre a mesma e avaliao da Instituio. Todas essas informaes obtidas em cada ano so disponibilizadas pelo INEP como microdadados.

Obtenção dos Dados¶

Para o desenvolvimento deste trabalho foram utilizados os microdados do Enade disponibilizados pelo site do INEP [2]. A pesquisa se baseou em dados de todos os estudantes que realizaram o exame no ano de 2017, 2018 e 2019. Dentre os dados disponveis esto:

  • Informaes do Curso, da Instituio de Ensino Superior e da sua localizao

  • Informaes Sociais do Estudante como gnero e idade

  • Avaliao - Formao Geral e Componente Especfico

  • Nota Geral e todas as notas que a compoem

  • Presena do Aluno no Enade e nas sees das provas

  • Gabarito das questes objetivas e as respostas marcadas pelo aluno

  • Questionrio do Estudante, com questes socioeconmicas e questes sobre sua vida acadmica

Totalizando 182 atributos na base de 2017 e 169 em cada umas das outras bases. Essa diferena se d por varveis que esto presentes apenas na base de 2017 referentes questes especficas do curso de licenciatura. Dessa forma, como esses atributos so especficos para um grupo de cursos e no esto presentes em todas as bases, dificultando assim uma integrao para uma anlise exploratria, foram descartados neste trabalho.

As informaes so disponibilizadas no formato TXT acompanhadas de um dicionrio de variveis no formato XLS que apresenta uma aba com a descrio do que significa cada atributo e seus possveis valores e outra aba com atributos identificadores de cada cdigo de cidade disponibilizados pelo IBGE, j que os microdados aprensentam apenas esses cdigos na identifio do local do curso.

Alm dos dados descritos acima, foram tambm utilizados nesse trabalho as bases tambm fornecidas pelo INEP que apresentam o Conceito ENADE de cada ano e podem ser encontradas na pgina do INEP [3]. Essas bases foram utilizadas com a finalidade de obter atributos referentes alguns cdigos que aparecem no microdados do ENADE como nome da instituio.

def download_arquivo(url, path_arquivo, nome_arquivo):
    
    # Cria a o pasta se no existe ainda no diretrio ainda
    if not os.path.exists(path_arquivo):
        os.makedirs(path_arquivo) 
    
    mensagem_download = "Realizando Download do Arquivo '{}'...".format(nome_arquivo)

    requisicao_arq = requests.get(url, path_arquivo + nome_arquivo)
    url_content = requisicao_arq.content
    csv = open(path_arquivo + nome_arquivo, 'wb')
    csv.write(url_content)
    csv.close()
    print(mensagem_sucesso, end="\n")

Modelagem de Dados¶

No presente trabalho foi utilizada a modelagem Estrela (Star Schema) caracterizada por poucas tabelas e relacionamentos, onde todas as tabelas de Dimenso se relacionam direta e unicamente com a tabela Fato, sendo assim um modelo simples e eficiente.

Tratamento de Dados¶

Para construo do modelo, o dataset passou foi avaliado e passou por tratamento de dados. O dataset apresenta algumas linhas que no apresentam notas das provas, escolhidas para serem fatos da modelagem e target do modelo, assim foi decidido pela remoo desses casos. Tambm foi optado pela remoo de nulos da coluna QI_01 j que foi oberservado que essas amostras tambm apresentavam valores nulos em quase todas as outras colunas pertencetes ao questionrio. Uma outra remoo foi a de amostras com idades duvidosas como 4 e 11 anos, assim foi criado um filtro para s incluir linhas que apresentam idade maior ou igual 16. Assim, aps todas as remes feitas, o dataset final ficou com 1.291.772 linhas.

Outro tratamento feito foi a substituio de valores pelos seus significados apresentados no dicionrio de dados nas colunas. Algumas colunas possuiam valores diferentes em cada ano que possuiam o mesmo significado, nesses casos, foram obtidas informaes de todos os dicionrios para que todos os valores fossem substudos, ocasionando numa mesma substituio para valores diferentes.

Todas essas manipulaes podem ser observadas no script python 01_Criao_Banco_de_Dados.ipynb na sesso onde as tabelas de dimenso e fato so construdas.

Banco de Dados¶

Alimentação do Banco de Dados¶

Para criao do banco de dados relacional deste trabalho foi escolhido a utilizao do sistema de gerenciamento sqlite por possuir uma gerenciamento mais simples atravs do python. Para alimentar o banco de dados com as tabelas criadas pela modelagem dimensional, foi utlizado o script 01_Criao_Banco_de_Dados.ipynb em python que utiliza a biblioteca sqlite3 para realizar a comunicao com o banco. Para isso, o script primeiro verifica se h o banco de dados Enade_DW.sqlite em seu diretrio, se no , o cria. Em seguida, realiza todas as manipulaes no dataset para criao das tabelas.

Com a conexo ao banco de dados j estabelecida e as tabelas criadas, para cada tabela, o script verfica se a tabela j existe no banco de dados, se no, a importa. Assim, se o script for executado com o banco de dados j alimentado, nada ser importado.

Modelo de Predição¶

Para esse trabalho foi elaborado um modelo de predio que com base alguns dos dados fornecidos pelo estudante (todos os selecionados para compor as dimenses do modelo estrela desenvolvido) classifique se o estudante tirou uma nota acima da nota mdia do ano em que prestou a prova. Primeiramente, calculou-se a nota mdia de cada ano utlizando toda a base, com o conhecimento desses valores, criou-se a coluna target que indica se a nota geral do estudante maior do que a nota mdia do ano, assim, se o estudante obteve uma nota maior do que a nota geral mdia do ano em que realizou a prova, a varivel ser igual 1, caso contrrio, ser 0. O modelo tem ento o objetivo de classificar essa varivel como 1 ou 0 para cada estudante, classificando assim seu desempenho.

O modelo foi implementado na linguagem python e pode ser visto no scrpit 03_Aprendizado_Random_Forest_Nota.ipynb.
